Azure App Service Web App Deployment - Project Summary
======================================================
Name: Pelumi
Date: June 2026

RUNTIME STACK
-------------
Node.js 22-LTS was selected as the runtime stack. It is the latest Long-Term Support 
version of Node.js, offering stability and modern JavaScript features. The application 
is a minimal HTTP server built using Node's built-in http module with no external 
dependencies.

PRICING TIER
------------
The Free F1 tier was selected for the App Service Plan (asp-webapp-lab-eastus). 
This tier provides 1 GB memory, shared compute resources, and 60 minutes of CPU 
time per day — sufficient for learning and testing purposes. It operates on Linux 
as the operating system, hosted in the West Europe region.

DEPLOYMENT METHOD
-----------------
The application was deployed by directly writing files to the /home/site/wwwroot 
directory via the Kudu SSH terminal. This approach was necessary due to a known 
issue with the Azure QuickDeploy (Preview) UI, which could not detect the Node.js 
platform version from the uploaded ZIP file. Placing app.js and package.json 
directly in the web root allowed Azure App Service to serve the application 
correctly after a container restart.

ENVIRONMENT VARIABLES CONFIGURED
---------------------------------
- NODE_ENV: production
- APP_VERSION: 1.0.0
- WELCOME_MESSAGE: Hello from Azure App Service

KEY LEARNINGS
-------------
- PaaS reduces operational overhead by abstracting infrastructure management.
- App Service Plans define the compute resources shared across hosted applications.
- Environment variables allow configuration without changing source code.
- Deployment slots (staging/production swap) require Standard tier or above.
- Azure Monitor provides built-in metrics including CPU Time and request counts.
- Horizontal scaling (Scale out) is disabled on the Free tier.
